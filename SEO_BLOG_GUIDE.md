# نظام كتابة المقالات المتوافقة مع السيو - دليل الاستخدام

## نظرة عامة

تم تطوير نظام متكامل لكتابة المقالات المتوافقة مع أحدث معايير السيو (SEO) لموقع كيان مزلات. النظام يتضمن:

- ✅ محرر نصوص غني (Rich Text Editor) مع دعم Markdown
- ✅ تحليل السيو التلقائي مع درجة تقييم (0-100)
- ✅ معاينة Meta Tags (Google/Twitter/Facebook)
- ✅ نظام التصنيفات والوسوم الكامل
- ✅ مساعد الذكاء الاصطناعي للكتابة
- ✅ صفحات عرض المدونة الكاملة
- ✅ أمان محسّن (تشفير كلمات المرور)
- ✅ Sitemap و Robots.txt تلقائي
- ✅ Structured Data (JSON-LD)

---

## الميزات الرئيسية

### 1. المحرر الغني (Rich Text Editor)

**الموقع:** لوحة الإدارة > المدونة > إضافة/تعديل مقال > تبويب "المحتوى"

**المميزات:**
- تنسيق النصوص (عريض، مائل، تسطير)
- العناوين (H1-H6)
- القوائم (مرقمة وغير مرقمة)
- الروابط مع عناوين
- الصور مع نص بديل
- الجداول
- أكواد برمجية مع تلوين
- دعم Markdown كامل
- دعم RTL للعربية

**كيفية الاستخدام:**
1. افتح لوحة الإدارة على `/admin/dashboard`
2. اختر "المدونة" من القائمة الجانبية
3. اضغط "إضافة مقال"
4. استخدم شريط الأدوات للتنسيق
5. يمكنك رفع الصور مباشرة من المحرر

---

### 2. تحليل السيو التلقائي

**الموقع:** تبويب "السيو" في محرر المقال

**ما يتم تحليله:**
- ✓ عدد الكلمات (الأمثل: 1500+)
- ✓ طول العنوان (50-60 حرف)
- ✓ طول الوصف التعريفي (150-160 حرف)
- ✓ كثافة الكلمة المفتاحية (1-2%)
- ✓ وجود الكلمة المفتاحية في العنوان
- ✓ وجود الكلمة المفتاحية في الفقرة الأولى
- ✓ بنية العناوين (H1, H2, H3)
- ✓ طول الفقرات
- ✓ الروابط الداخلية والخارجية
- ✓ الصور مع النص البديل
- ✓ سهولة القراءة
- ✓ بنية الرابط (URL)

**الدرجات:**
- 80-100: ممتاز (أخضر)
- 60-79: جيد (أصفر)
- 0-59: يحتاج تحسين (أحمر)

**نصائح:**
- اكتب محتوى طويل (1500+ كلمة)
- استخدم الكلمة المفتاحية بشكل طبيعي
- أضف روابط داخلية لمقالات أخرى
- استخدم عناوين فرعية منظمة
- أضف نصاً بديلاً لجميع الصور

---

### 3. معاينة Meta Tags

**الموقع:** تبويب "السيو" في محرر المقال

**المعاينات المتاحة:**
- Google Search Results
- Twitter Card
- Facebook OpenGraph

**الحقول:**
- **Meta Title:** عنوان مخصص لمحركات البحث (50-60 حرف)
- **Meta Description:** وصف مخصص (150-160 حرف)
- **Focus Keyword:** الكلمة المفتاحية الرئيسية
- **Keywords:** كلمات مفتاحية إضافية (مفصولة بفواصل)

---

### 4. مساعد الذكاء الاصطناعي

**الموقع:** تبويب "مساعد AI" في محرر المقال

**الوظائف المتاحة:**

#### أ. التوليد (Generate)
- **توليد عناوين مقترحة:** 8 عناوين محسّنة للسيو
- **توليد مخطط المقالة:** بنية كاملة مع عناوين رئيسية وفرعية
- **توليد المقدمة:** مقدمة جذابة (100-150 كلمة)
- **توليد الخاتمة:** خاتمة قوية مع دعوة لاتخاذ إجراء
- **توسيع قسم معين:** كتابة محتوى تفصيلي لقسم محدد
- **توليد الوصف التعريفي:** وصف meta محسّن (150-160 حرف)

#### ب. التحسين (Improve)
- **تحسين الوضوح:** إعادة صياغة للوضوح
- **تحسين القراءة:** جمل أقصر وأبسط
- **تحسين السيو:** تحسين استخدام الكلمات المفتاحية
- **تصحيح النحو:** إصلاح الأخطاء النحوية والإملائية

#### ج. الكلمات المفتاحية (Keywords)
- **اقتراح كلمات مفتاحية:** 10 كلمات مفتاحية ذات صلة (LSI Keywords)

**كيفية الاستخدام:**
1. أدخل الكلمة المفتاحية الرئيسية أولاً
2. (اختياري) أضف موجز المحتوى
3. اختر الوظيفة المطلوبة
4. سيتم إدراج المحتوى المولد تلقائياً

---

### 5. نظام التصنيفات والوسوم

#### التصنيفات (Categories)
**الموقع:** لوحة الإدارة > إدارة التصنيفات

**الحقول:**
- الاسم (مطلوب)
- الرابط (Slug) - يتم توليده تلقائياً
- الوصف
- الصورة

**الاستخدام:**
- كل مقالة يمكن أن تنتمي لتصنيف واحد فقط
- التصنيفات تظهر في صفحة المدونة كفلاتر
- لكل تصنيف صفحة أرشيف خاصة: `/blog/category/[slug]`

#### الوسوم (Tags)
**الموقع:** لوحة الإدارة > إدارة الوسوم

**الحقول:**
- الاسم (مطلوب)
- الرابط (Slug) - يتم توليده تلقائياً

**الاستخدام:**
- كل مقالة يمكن أن تحتوي على عدة وسوم
- الوسوم تظهر كسحابة وسوم
- لكل وسم صفحة أرشيف خاصة: `/blog/tag/[slug]`

---

### 6. صفحات المدونة

#### أ. صفحة القائمة (`/blog`)
**المميزات:**
- عرض شبكي للمقالات (12 مقالة لكل صفحة)
- بحث في المقالات
- فلترة حسب التصنيف
- شريط جانبي بالتصنيفات
- ترقيم الصفحات (Pagination)

#### ب. صفحة المقال (`/blog/[slug]`)
**المميزات:**
- عرض المحتوى الكامل مع تنسيق Markdown
- جدول المحتويات (TOC) تلقائي
- معلومات المؤلف والتاريخ
- وقت القراءة وعدد المشاهدات
- أزرار المشاركة (واتساب، تويتر، فيسبوك، لينكدإن)
- مقالات ذات صلة (نفس التصنيف)
- التنقل للمقال السابق/التالي
- Breadcrumbs للتنقل
- JSON-LD Structured Data

#### ج. صفحات الأرشيف
- `/blog/category/[slug]` - مقالات التصنيف
- `/blog/tag/[slug]` - مقالات الوسم

### 7. الأمان

**التحسينات المطبقة:**
- ✅ تشفير كلمات المرور باستخدام bcrypt
- ✅ Middleware لحماية مسارات الإدارة
- ✅ Cookies آمنة (httpOnly, secure)
- ✅ Session tokens مع انتهاء صلاحية (7 أيام)

**ملاحظة مهمة:**
كلمات المرور الحالية في قاعدة البيانات بنص صريح. يجب تحديثها:
1. سجل دخول للوحة الإدارة
2. اذهب إلى الإعدادات
3. غيّر كلمة المرور
4. سيتم تشفيرها تلقائياً

### 8. السيو التقني

#### Sitemap (`/sitemap.xml`)
يتم توليده تلقائياً ويتضمن:
- الصفحة الرئيسية
- صفحة المدونة
- جميع المقالات المنشورة
- جميع صفحات التصنيفات
- جميع صفحات الوسوم

#### Robots.txt (`/robots.txt`)
- يسمح لجميع محركات البحث
- يمنع الوصول لمسارات الإدارة (`/admin`, `/api/admin`)
- يشير إلى موقع Sitemap

#### Structured Data (JSON-LD)
- Article Schema لكل مقالة
- Organization Schema
- BreadcrumbList Schema
- WebSite Schema مع SearchAction

## دليل الاستخدام السريع

### إنشاء مقال جديد

1. **افتح لوحة الإدارة**
   - اذهب إلى `/admin/dashboard`
   - سجل دخول

2. **اختر المدونة**
   - من القائمة الجانبية، اختر "المدونة"

3. **أضف مقال جديد**
   - اضغط "إضافة مقال"

4. **املأ المعلومات الأساسية** (تبويب المحتوى)
   - العنوان
   - المقتطف (ملخص قصير)
   - المحتوى (استخدم المحرر الغني)
   - الصورة الرئيسية + النص البديل

5. **حسّن السيو** (تبويب السيو)
   - أدخل الكلمة المفتاحية الرئيسية
   - اكتب Meta Title (50-60 حرف)
   - اكتب Meta Description (150-160 حرف)
   - راجع درجة السيو والاقتراحات
   - عاين كيف ستظهر في محركات البحث

6. **استخدم مساعد AI** (تبويب مساعد AI) - اختياري
   - ولّد عناوين مقترحة
   - ولّد مخطط المقالة
   - حسّن المحتوى
   - اقترح كلمات مفتاحية

7. **اضبط الإعدادات** (تبويب الإعدادات)
   - اختر التصنيف
   - أضف الوسوم
   - حدد المؤلف
   - فعّل "نشر المقال" عند الجاهزية

8. **احفظ**
   - اضغط "إنشاء" أو "تحديث"

## نصائح لتحسين السيو

### 1. المحتوى
- اكتب مقالات طويلة (1500+ كلمة)
- استخدم الكلمة المفتاحية في العنوان والفقرة الأولى
- قسّم المحتوى بعناوين فرعية واضحة
- استخدم فقرات قصيرة (150 كلمة أو أقل)
- أضف أمثلة عملية وقوائم

### 2. الكلمات المفتاحية
- كثافة مثالية: 1-2%
- استخدمها بشكل طبيعي
- أضف كلمات مفتاحية ذات صلة (LSI)
- ضعها في العناوين الفرعية

### 3. الروابط
- أضف 2-3 روابط داخلية لمقالات أخرى
- أضف رابط خارجي واحد على الأقل لمصدر موثوق
- استخدم نصوص رابط وصفية

### 4. الصور
- أضف نصاً بديلاً (alt text) لكل صورة
- استخدم أسماء ملفات وصفية
- ضغط الصور قبل الرفع

### 5. Meta Tags
- Meta Title: 50-60 حرف، يحتوي على الكلمة المفتاحية
- Meta Description: 150-160 حرف، جذاب ويشجع على النقر
- استخدم كلمات قوية (دليل، شامل، أفضل، 2026)

## قاعدة البيانات

### الجداول الجديدة

#### BlogPost (محدّث)
```prisma
model BlogPost {
  // الحقول الأساسية
  id          String
  title       String
  slug        String @unique
  excerpt     String?
  content     String
  image       String?
  author      String
  published   Boolean
  
  // حقول السيو الجديدة
  metaTitle         String?
  metaDescription   String?
  focusKeyword      String?
  keywords          String?
  readingTime       Int?
  wordCount         Int?
  viewCount         Int @default(0)
  seoScore          Int?
  featuredImageAlt  String?
  
  // التواريخ
  publishedAt   DateTime?
  scheduledAt   DateTime?
  createdAt     DateTime
  updatedAt     DateTime
  
  // العلاقات
  categoryId    String?
  category      Category?
  tags          Tag[]
}
```

#### Category (جديد)
```prisma
model Category {
  id          String
  name        String
  slug        String @unique
  description String?
  image       String?
  posts       BlogPost[]
  createdAt   DateTime
  updatedAt   DateTime
}
```

#### Tag (جديد)
```prisma
model Tag {
  id        String
  name      String
  slug      String @unique
  posts     BlogPost[]
  createdAt DateTime
}
```

## الدعم والمساعدة

إذا واجهت أي مشاكل:

1. **تحقق من السجلات (Logs)**
   - افتح Console في المتصفح (F12)
   - راجع سجلات الخادم في Terminal

2. **المشاكل الشائعة**
   - **المحرر لا يظهر:** تأكد من تثبيت `@mdxeditor/editor`
   - **الصور لا ترفع:** تحقق من مجلد `/public/uploads/blog/`
   - **درجة السيو 0:** تأكد من ملء الكلمة المفتاحية والمحتوى

3. **إعادة تشغيل الخادم**
   ```bash
   # أوقف الخادم (Ctrl+C)
   # ثم شغله مرة أخرى
   npm run dev
   ```
